## 1. 文档目标

本文件用于将第一期现有方案收敛为一份统一需求文档，作为以下工作的共同基线：

- 高保真原型设计
- 前端页面开发
- 后端接口与数据能力开发
- 联调、测试与验收

本期目标不是做完整投研平台，而是先完成“研究信息中台 MVP”闭环，让系统具备从文档接入、知识检索、研究问答到研究资产沉淀的可用能力。

---

## 2. 一期建设目标

### 2.1 业务目标

- 支持研究资料的统一上传、解析、切分和入库
- 支持基于知识库的检索与带引用问答
- 支持生成研究备忘录与晨会日报
- 支持将生成内容沉淀为可编辑、可导出的研究资产

### 2.2 产品目标

- 提供一套可直接演示和评审的高保真原型
- 提供一套前后端边界清晰、可联调的功能方案
- 保证一期范围闭环，不引入二期、三期复杂能力

### 2.3 一期成功标准

- 用户可完成单文档上传并触发入库
- 用户可围绕指定公司发起检索与问答，并查看引用来源
- 用户可生成研究备忘录和晨会日报，并保存为研究资产
- 用户可在前端完成文档、任务、资产的查看和基础管理

## 4. 目标用户与核心场景

### 4.1 目标用户

- 投研分析师：上传研报、问答、生成备忘录
- 研究负责人：查看日报、复核资产内容
- 系统维护者：查看基础配置和运行状态

### 4.2 核心场景

#### 场景 A：上传研报并提问

用户上传研报后，系统完成解析、切分、向量化和入库；用户随后发起问题，系统返回带引用的答案。

#### 场景 B：围绕公司生成研究备忘录

用户选择公司与主题，系统检索历史资料并生成结构化备忘录；用户在前端编辑后保存为研究资产。

#### 场景 C：生成晨会日报

用户输入日期或主题，系统汇总研究资料并输出日报草稿；用户复核后导出 Markdown 或 DOCX。

## 5. 高保真原型要求

### 5.1 原型目标

一期原型不是线框图，而是可直接用于评审、走查和开发对照的高保真原型。原型需完整体现页面结构、组件层级、核心交互、状态反馈和信息密度。

### 5.2 原型交付范围

- 全局布局与导航
- 核心页面完整视觉稿
- 关键弹窗、抽屉、表单
- 空状态、加载状态、失败状态
- 关键交互流程串联
- 响应式布局说明

### 5.3 原型设计基线

- 风格定位：专业、沉稳、高效
- 默认主题：深色金融主题
- 页面结构：左侧导航 + 顶部栏 + 主内容区
- 信息层次：高密度表格、卡片、状态标签并存
- 交互反馈：上传进度、任务状态、流式输出、保存结果、错误提示需可视化

### 5.4 原型必须覆盖的状态

- 初始态
- 加载态
- 空数据态
- 成功态
- 失败态
- 长文本滚动态
- 流式输出态
- 删除确认态

### 5.5 原型交付物清单

- 站点地图
- 页面流程图
- 页面高保真视觉稿
- 组件规范页
- 页面状态页
- 关键交互说明

## 7. 前端需求

### 7.1 前端技术基线

- 框架：Next.js 14（App Router）
- 语言：TypeScript
- 样式：Tailwind CSS
- 组件：shadcn/ui
- 状态管理：Zustand
- 表单校验：React Hook Form + Zod
- 数据通信：REST API + SSE

### 7.2 全局前端能力

- 支持侧边栏导航与面包屑
- 支持统一分页、筛选、状态标签
- 支持全局 Toast 提示
- 支持骨架屏与空状态
- 支持任务轮询
- 支持问答流式输出
- 支持 Markdown 内容展示与编辑

### 7.3 页面需求清单

#### 7.3.1 轻量首页 `/dashboard`

功能要求：

- 展示快捷入口：上传文档、智能问答、生成备忘录、生成日报
- 展示最近任务列表
- 展示最近研究资产列表
- 可展示系统基础信息

交互要求：

- 点击快捷入口跳转对应页面
- 最近任务点击后可进入任务中心或相关详情
- 最近资产点击后可进入资产详情页

#### 7.3.2 文档管理 `/documents`

功能要求：

- 单文档上传
- 文档列表展示
- 按文档类型、公司、状态、关键词、日期筛选
- 查看文档详情
- 删除文档

页面模块：

- 顶部操作区
- 筛选栏
- 文档表格
- 上传弹窗

交互要求：

- 上传表单至少包含文件、文档类型，支持补充标题、来源、公司、发布日期、权限级别
- 上传成功后支持触发入库
- 列表支持分页
- 状态需显示 `pending / processing / parsed / failed`

#### 7.3.3 文档详情 `/documents/[id]`

功能要求：

- 展示文档元信息
- 展示解析状态
- 展示文本预览
- 展示 chunk 列表
- 支持再次入库

交互要求：

- 支持查看原文摘要预览
- 支持分页查看 chunk
- 对失败状态提供重试入口

#### 7.3.4 智能问答 `/qa`

功能要求：

- 输入问题并发起问答
- 选择公司范围
- 展示流式回答
- 展示引用卡片
- 支持生成研究备忘录
- 支持生成晨会日报

交互要求：

- 问答结果需按“回答区域 + 引用区域”布局
- 流式输出期间需展示生成中状态
- 引用需可展示文档标题、片段摘要、页码或 chunk 信息
- 对问答失败、无结果、模型异常要有明确提示

#### 7.3.5 任务中心 `/tasks`

功能要求：

- 展示任务列表
- 展示任务状态
- 查看任务详情
- 查看任务执行记录

交互要求：

- 支持自动轮询运行中任务
- 支持按状态筛选
- 任务详情需展示类型、发起时间、完成时间、错误信息

#### 7.3.6 研究资产 `/assets`

功能要求：

- 展示资产列表
- 支持按类型、状态、公司、关键词筛选
- 查看资产详情
- 编辑资产内容
- 查看版本记录
- 导出 Markdown / DOCX

交互要求：

- 编辑器支持 Markdown
- 保存后需生成修订记录
- 导出动作需给出结果反馈

#### 7.3.7 系统设置 `/settings`

功能要求：

- 展示系统基础信息
- 展示 Prompt 模板列表
- 支持查看模板详情

一期要求收缩为只读或轻编辑模式，不纳入复杂系统参数管理。

## 9. 前后端联动要求

### 9.1 文档上传与入库链路

前端完成文件上传后，后端返回文档 ID；前端再触发入库接口，并在列表页和详情页展示处理状态。

### 9.2 问答链路

前端调用问答接口或 SSE 接口，后端返回回答内容、引用信息、任务信息；前端需分区展示答案和引用。

### 9.3 资产生成链路

前端通过“生成备忘录 / 生成日报”动作发起后端生成接口；后端返回资产内容与资产 ID；前端进入编辑态继续保存和导出。

### 9.4 任务回显链路

所有生成型任务应尽可能落任务记录；前端在任务中心统一查看执行状态。

## 14. 附录：一期范围边界说明

为了保证第一期可交付，本期坚持“中台闭环优先、复杂分析后置”的原则。

一期的核心不是做投资决策，而是先把以下闭环做扎实：

- 资料进来
- 知识可检索
- 问题可回答
- 内容可生成
- 结果可保存

二期、三期涉及的量化、回测、风控、组合能力，仅在导航和架构层面预留，不纳入本期设计和开发验收。
