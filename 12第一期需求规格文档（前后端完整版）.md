## 1. 项目概述

### 1.1 项目目标

构建一个**研究信息中台 MVP**，具备以下核心能力闭环：

```
文档接入 → 文档解析 → 知识检索 → AI 问答 → 备忘录/日报生成 → 研究资产沉淀
```

### 1.2 一期范围

| 功能模块 | 范围 | 状态 |
|---------|------|------|
| 文档上传 | PDF / DOCX / TXT / Markdown 上传 | ✅ P0 已实现 |
| 文档解析 | PyMuPDF + python-docx 解析，支持页码/段落定位 | ✅ P0 已实现 |
| 文档切分 | 可配置 chunk_size/chunk_overlap，段落级切分 | ✅ P0 已实现 |
| 向量化 | Ollama 本地 bge-m3 (1024维) embedding | ✅ P0 已实现 |
| 混合检索 | 向量检索 + 关键词检索 + 结果融合 | ✅ P0 已实现 |
| RAG 问答 | 带引用的研究问答，支持同步 + SSE 流式 | ✅ P0 已实现 |
| 总控 Agent | 任务意图识别、路由分发、结果汇总 | ✅ P1 已实现 |
| 研究 Agent | 研报检索、摘要、问答、备忘录生成 | ✅ P1 已实现 |
| 报告协作 Agent | 日报模板、Markdown/DOCX 导出 | ✅ P1 已实现 |
| 研究资产 | 备忘录/日报保存、版本管理、引用关联 | ✅ P1 已实现 |
| 标签体系 | 文档/资产的标签管理 | ✅ P1 已实现 |
| 公司主数据 | 公司代码、行业、市场等基础数据 | ✅ P1 已实现 |
| Prompt 模板 | 配置化的 Prompt 管理 | ✅ P1 已实现 |
| 前端工作台 | 文档管理/智能问答/研究资产/任务中心 | 🔲 P2 待开发 |

### 1.3 明确不做

- 因子计算、回测引擎、组合优化
- 风控监控 (VaR/合规校验)
- 交易系统对接
- 多租户、细粒度权限
- 本地大模型路由、代码沙箱

### 1.4 当前完成度

| 维度 | 进度 |
|------|------|
| 后端 API | ~95% (35个端点已实现) |
| 数据库 | 100% (15张表、82个索引) |
| LLM/Embedding | 100% (DeepSeek + Ollama bge-m3) |
| 核心 RAG 闭环 | 100% (检索→生成→引用已打通) |
| 前端 | 0% (待开发) |
| 测试覆盖 | ~7% (待补充) |

## 2. 系统架构

### 2.1 技术栈总览

| 层级 | 选型 | 说明 |

### 3.5 外键删除策略

| 关系 | 策略 | 原因 |
|------|------|------|
| documents → document_chunks | CASCADE | 文档物理删除后 chunk 无独立意义 |
| documents → document_tags | CASCADE | 关系随文档清理 |
| tasks → task_runs | CASCADE | 任务删除后明细同步清理 |
| tasks → research_assets | SET NULL | 资产需保留，来源任务可置空 |
| research_assets → asset_revisions | CASCADE | 版本记录随资产清理 |
| documents → citations | CASCADE | 引用随文档清理 |
| document_chunks → citations | SET NULL | 引用可保留快照，chunk 指针置空 |

> 注意：一期业务删除均为软删除（写入 `deleted_at`），以上策略仅在物理删除时生效。

## 4. 后端 API 规范

### 4.1 通用规范

#### API 前缀

业务接口统一使用 `/api/v1` 前缀。`GET /health` 作为负载均衡探活别名。

#### 统一响应格式

```json
// 成功
{
  "code": 0,
  "message": "ok",
  "data": {}
}

// 失败
{
  "code": 2003,
  "message": "document parse failed",
  "data": null,
  "request_id": "req_20260604_0001"
}
```

#### 分页规范

请求参数: `page`(默认1), `page_size`(默认20,最大100), `sort_by`, `sort_order`(asc/desc)

```json
{
  "items": [],
  "pagination": {
    "page": 1,
    "page_size": 20,
    "total": 100,
    "total_pages": 5,
    "has_next": true,
    "has_prev": false
  }
}
```

#### 错误码

| 错误码 | 说明 |
|--------|------|
| 1001 | 系统内部错误 |
| 1002 | 配置不存在 |
| 1003 | 外部模型服务不可用 |
| 2001 | 文档不存在 |
| 2002 | 文档类型不支持 |
| 2003 | 文档解析失败 |
| 2004 | 文档已存在 |
| 2005 | 文档未完成入库 |
| 3001 | 检索参数错误 |
| 3002 | 向量模型不可用 |
| 3003 | 检索无结果 |
| 3004 | LLM 生成失败 |
| 4001 | 任务不存在 |
| 4002 | 任务类型不支持 |
| 4003 | 任务执行失败 |
| 5001 | 资产不存在 |
| 5002 | 资产版本冲突 |
| 5003 | 资产导出失败 |
| 6001 | 公司不存在 |
| 7001 | 标签不存在 |

#### 链路追踪

- 客户端通过 Header `X-Request-ID` 传入请求 ID
- 若未传，服务端自动生成
- 错误响应中的 `request_id` 与该 Header 对齐
- 后端日志、`tasks.request_id`、`task_runs` 日志均记录同一 ID

### 4.3 核心调用链

#### 文档上传与入库链路

```
POST /api/v1/documents
  → DocumentService.create_document()
  → 写入 documents 表 (parse_status=pending)

POST /api/v1/documents/{id}/ingest
  → TaskService.create_task(task_type=document_ingest)
  → DocumentService.ingest_document()
  → DocumentParseSkill → 提取文本
  → ChunkingSkill → 切分为 chunks
  → EmbeddingSkill → 生成 1024 维向量
  → 批量写入 document_chunks (含 embedding)
  → 更新 documents.parse_status=parsed
  → TaskService.finish_task()
```

#### 问答链路

```
POST /api/v1/qa/ask
  → TaskService.create_task(task_type=qa)
  → OrchestratorAgent.run()
    → 意图识别: task_type=qa
    → 路由: ResearchAgent
  → ResearchAgent.run()
    → HybridRetrievalSkill → 向量+关键词混合召回
    → CitationTraceSkill → 收集引用信息
    → LLM (DeepSeek) → 基于上下文生成回答 [1][2]
  → CitationService.save() → 写入 citations 表
  → TaskService.finish_task()
```

#### 研究备忘录链路

```
POST /api/v1/qa/memo
  → TaskService.create_task(task_type=memo_generate)
  → OrchestratorAgent → ResearchAgent
  → HybridRetrievalSkill → 多维度检索
  → MemoGenerationSkill → 基于模板生成草稿
  → CitationTraceSkill → 引用追踪
  → 返回 content_markdown + citations
  → 用户修订后 → POST /api/v1/assets → 保存为资产
```

#### 晨会日报链路

```
POST /api/v1/qa/daily-report
  → TaskService.create_task(task_type=daily_report)
  → OrchestratorAgent → ReportingAgent
  → DailyReportSkill → 汇总市场/公告/新闻
  → HybridRetrievalSkill → 检索相关研究
  → 返回 content_markdown + citations
  → 用户复核 → POST /api/v1/assets → 保存资产
  → POST /api/v1/assets/{id}/export → 导出 DOCX
```

## 6. 核心业务流程

### 6.1 文档上传与解析流程

```
用户点击"上传文档"
  → 弹出上传对话框（支持拖拽）
  → 填写元数据（类型必填，其余可选）
  → 点击确认
  → 文件上传 → POST /api/v1/documents
  → 返回 document_id，跳转到文档详情页
  → 详情页显示"解析中..."（骨架屏 + 进度提示）
  → 自动触发入库 → POST /api/v1/documents/{id}/ingest
      → 后端: DocumentParseSkill → ChunkingSkill → EmbeddingSkill
  → 轮询解析状态（3s 间隔，检查 parse_status）
  → 解析完成 → 显示"已解析" + chunk 数量
  → 用户可查看 chunk 详情和原文预览
```

### 6.2 智能问答流程

```
用户打开问答页
  → 右侧设置公司筛选和返回数量（可选）
  → 输入问题
  → 发起流式问答 → POST /api/v1/qa/ask/stream
  → SSE 事件处理:
      metadata: 显示任务信息（task_id, provider）
      citation: 收集引用，右侧面板实时显示
      delta: 逐字追加到回答区域（打字机效果）
      done: 完整答案展示，显示操作按钮（生成备忘录、复制）
      error: 显示错误提示
  → 用户操作:
      点击引用编号 → 右侧面板定位到对应 chunk
      "生成备忘录" → 跳转备忘录编辑页（预填问答内容和引用）
      "复制答案" → 复制 Markdown 格式答案
```

### 6.3 研究备忘录生成流程

```
方式1：从问答页快捷入口
  → 问答结果点击"生成备忘录"
  → 跳转备忘录编辑页，自动填充标题和内容

方式2：从资产页新建
  → 选择公司、输入标题和查询重点
  → POST /api/v1/qa/memo
  → 后端: OrchestratorAgent → ResearchAgent → MemoGenerationSkill
  → 等待生成完成（同步返回或轮询任务状态）
  → 进入编辑器，显示 Markdown 内容

编辑阶段：
  → 支持直接编辑 Markdown（双栏预览）
  → 可插入/删除引用编号
  → 保存为草稿 → POST /api/v1/assets（version=1, status=draft）
  → 编辑后保存 → PUT /api/v1/assets/{id}（自动递增 version）
  → 发布 → 更改 status 为 published
  → 导出 → POST /api/v1/assets/{id}/export（format=docx）
```

### 6.4 晨会日报生成流程

```
从资产页或首页快捷入口
  → 选择日期、包含公司、是否包含市场概要
  → POST /api/v1/qa/daily-report
  → 后端: OrchestratorAgent → ReportingAgent → DailyReportSkill
      → HybridRetrievalSkill 检索相关公告/研报
      → LLM 基于模板汇总生成日报
  → 返回 Markdown 内容 + 引用列表
  → 进入编辑器 → 复核 → 修改 → 保存 → 导出
```

### 6.5 总控 Agent 路由逻辑

```
POST /api/v1/tasks (task_type)
  → OrchestratorAgent
      ├── task_type=qa              → ResearchAgent → QA生成
      ├── task_type=memo_generate   → ResearchAgent → 备忘录生成
      ├── task_type=daily_report    → ReportingAgent → 日报生成
      └── task_type=document_ingest → 直接执行 Skill 链（不走 Agent 路由）
  → 汇总结果
  → 写入 tasks (status + output_payload + result_summary)
  → 写入 citations (如有)
```

### 7.2 后端待修复项（P0）

| 问题 | 影响 | 修复方案 |
|------|------|----------|
| 无连接池 | 每次请求新连接，性能差 | 用 asyncpg 连接池替换 psycopg2 直连 |
| save_citations 缺事务 | 引用写入不完整时无回滚 | 加 conn.transaction() 包裹 |
| LLM/Embedding 无重试 | 偶发超时直接失败 | 加 tenacity 重试装饰器 |
