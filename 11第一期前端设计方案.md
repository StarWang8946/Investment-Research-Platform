## 1. 技术架构

### 1.1 核心技术栈

| 模块 | 选型 | 版本 | 选型理由 |
|------|------|------|----------|
| 框架 | Next.js | 14.x (App Router) | SSR/CSR 灵活切换，文件路由，API Routes 可做 BFF |
| 语言 | TypeScript | 5.x | 类型安全，减少运行时错误，提升开发效率 |
| UI 框架 | Tailwind CSS | 3.x | 原子化 CSS，与深色金融主题适配好，自定义成本低 |
| 组件库 | shadcn/ui | - | 按需引入，不依赖第三方运行时；源码可控 |
| 图标 | Lucide React | 0.294+ | 轻量、一致性好、社区活跃 |
| HTTP | Axios | 1.x | 拦截器统一错误处理、请求取消、SSE 适配 |
| 日期 | date-fns | 3.x | 函数式、轻量、tree-shaking 友好 |
| Markdown 渲染 | react-markdown + remark-gfm | - | 备忘录/日报内容展示 |
| 流式响应 | EventSource / fetch SSE | - | 问答流式输出 |
| 状态管理 | Zustand | - | 轻量、无 boilerplate、支持持久化、TypeScript 友好 |
| 表单 | React Hook Form + Zod | - | 性能好、校验与类型一体化 |

### 1.4 关键架构决策

| 决策 | 方案 | 理由 |
|------|------|------|
| 路由模式 | App Router (文件路由) | Next.js 14 推荐，支持 Server Components，布局嵌套自然 |
| 渲染策略 | CSR 为主 | 投研平台是登录后使用的 SPA 场景，CSR 体验更流畅 |
| 状态管理 | Zustand | 比 Redux 轻量 90%，无 boilerplate，支持持久化 |
| API 通信 | REST + SSE | REST 处理 CRUD；SSE 处理流式问答；一期不引入 GraphQL |
| 组件策略 | shadcn/ui 按需引入 | 不依赖第三方运行时，源码可控，样式统一 |
| 图表 | 一期不引入 | 一期核心页面不依赖复杂图表，二期量化页面启用 Recharts |

### 2.2 一期明确不做

| 功能 | 原因 |
|------|------|
| 问答历史会话恢复 | 需要会话上下文存储，一期最小闭环不包含 |
| 批量入库 UI | 后端有接口，但一期前端优先单文档流程 |
| 批量删除 UI | 后端有接口，但一期优先单文档操作 |
| 版本回滚专用按钮 | 后端接口 P2，一期可先展示版本列表 |
| 全局搜索 `Cmd/Ctrl + K` | 快捷键体系延后，避免焦点冲突 |
| 系统参数可视化编辑 | 后端 configs 接口 P3，一期只读展示 |
| 登录/权限体系 | 一期本地单机使用，不设权限 |
| 浅色主题 | 一期只做深色主题，浅色预留变量但不验收 |
| 复杂金融图表 | 一期无量化页面，图表组件预留目录 |

## 3. UI 风格设计

### 3.1 设计定位

**"专业、沉稳、高效"** — 面向专业投研人员的金融级界面

- **专业感**：深色主题为主，数据密度高，信息层次清晰
- **沉稳感**：低饱和度配色，避免花哨动画，聚焦内容
- **高效感**：操作路径短，批量操作，搜索先行

### 3.7 动效规范

| 类型 | 时长 | 曲线 | 场景 |
|------|------|------|------|
| 微交互 | 150ms | ease | 按钮 hover、输入框聚焦 |
| 过渡 | 200ms | ease-in-out | 面板展开、Tab 切换 |
| 弹出 | 250ms | cubic-bezier(0.16,1,0.3,1) | 弹窗出现、下拉展开 |
| 数据加载 | 骨架屏 | - | 列表、表格加载 |
| 流式输出 | 打字机效果 | - | 问答流式输出 |

**不做的动效**：粒子效果、3D 翻转、大幅度位移动画 — 投研场景追求效率，动效以功能性为主。

## 6. 交互说明

### 6.1 核心交互流程

#### 6.1.1 文档上传与入库流程

```
用户点击"上传文档"
  → 弹出上传对话框（支持拖拽）
  → 填写元数据（类型必填，其余可选）
  → 点击确认
  → 文件上传 → POST /api/v1/documents
  → 返回 document_id，跳转到文档详情页
  → 详情页显示"解析中..."（骨架屏 + 进度提示）
  → 自动触发入库 → POST /api/v1/documents/{id}/ingest
  → 轮询解析状态（3s 间隔）
  → 解析完成 → 显示"已解析" + chunk 数量
  → 用户可查看 chunk 预览
```

#### 6.1.2 智能问答流程

```
用户打开问答页
  → 右侧设置公司筛选和返回数量（可选）
  → 输入问题
  → 发起流式问答 → POST /api/v1/qa/ask/stream
  → SSE 事件处理：
    · metadata: 显示任务信息
    · citation: 收集引用，右侧面板实时显示
    · delta: 逐字追加到回答区域（打字机效果）
    · done: 完整答案展示，显示操作按钮
    · error: 显示错误提示
  → 用户操作：
    · 点击引用编号 → 右侧面板定位到对应 chunk
    · "生成备忘录" → 跳转备忘录编辑页（预填问答内容）
    · "复制答案" → 复制 Markdown 格式答案
```

#### 6.1.3 研究备忘录生成流程

```
方式1：从问答页快捷入口
  → 问答结果点击"生成备忘录"
  → 跳转备忘录编辑页，自动填充标题和内容

方式2：从资产页新建

  → 等待生成完成
  → 进入编辑器，显示 Markdown 内容

  → 可插入/删除引用
  → 保存为草稿 → POST /api/v1/assets

  → 导出 → POST /api/v1/assets/{id}/export
```

### 7.2 SSE 流式处理（修正版）

```typescript
// hooks/useSSE.ts
// 后端实际 SSE 事件：metadata / citation / delta / done / error

function useSSE(url: string, body: any) {
  const [data, setData] = useState('')
  const [citations, setCitations] = useState([])
  const [status, setStatus] = useState('idle')

  const start = async () => {
    const response = await fetch(url, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(body),
    })

    const reader = response.body!.getReader()
    const decoder = new TextDecoder()

    while (true) {
      const { done, value } = await reader.read()
      if (done) break

      const text = decoder.decode(value)
      // 解析 SSE 事件：metadata / citation / delta / done / error
      parseSSEEvents(text, {
        onMetadata: (m) => {
          setStatus('generating')
          // 保存 task_id 等元信息
        },
        onCitation: (c) => setCitations(prev => [...prev, c]),
        onDelta: (delta) => setData(prev => prev + delta.text),
        onDone: () => setStatus('done'),
        onError: (e) => {
          setStatus('error')
          showToast(e.message, 'error')
        },
      })
    }
  }

  return { data, citations, status, start }
}
```
